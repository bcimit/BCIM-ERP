0.  SYSTEM OVERVIEW & GOLDEN THREAD PHILOSOPHY
================================================================================

ConstructERP v3.0 is a forensic intelligence platform built for the
construction industry. The core philosophy is called the "Golden Thread":

  A single measurement recorded by a site engineer flows downstream to:
    -> Update the Client Bill (RA Bill)
    -> Update the Material Reconciliation (Theoretical vs Actual)
    -> Update the Labor Efficiency Index
    -> Update the Corporate Profit & Loss (Command Hub)

DATA IS NEVER ENTERED TWICE. Every module reads from the same central
database. This eliminates reconciliation errors and ensures 100% auditability.

TECHNOLOGY STACK:
  - Frontend: React.js with Framer Motion animations
  - Backend:  Node.js / Express.js with PostgreSQL database
  - Auth:     JWT tokens with 1-hour expiry + 60-minute idle timeout
  - Hosting:  Self-hosted or cloud-deployed

1.  LOGIN, AUTHENTICATION & SESSION SECURITY
================================================================================

URL: /login

WHAT YOU SEE:
  - Email address input field
  - Password input field
  - "Sign In" button
  - Link to Register page

PASSWORD POLICY (enforced on registration):
  - Minimum 8 characters
  - Must contain at least 1 uppercase letter (A-Z)
  - Must contain at least 1 lowercase letter (a-z)
  - Must contain at least 1 number (0-9)
  - Must contain at least 1 special character (@$!%*?&)

SESSION SECURITY:
  - JWT Access Token expires after 1 hour
  - Global Idle Timeout: If no mouse movement or keyboard activity is
    detected for 60 minutes, the user is automatically logged out
  - Password hashing uses bcrypt with 12 salt rounds

REGISTRATION (URL: /register):
  Fields required:
    - Full Name
    - Email Address
    - Password (must meet complexity rules above)
    - Confirm Password (must match)

DIGITAL SIGNATURE:
  - After logging in, users should go to their Profile page
  - Upload a PNG/JPG image of their handwritten signature
  - This signature appears on all printed documents (MRS, PO, GRN, etc.)
  - Without a signature, approval actions are still possible but a
    warning banner appears: "Identity Verification Required"

----------------------
4.4  MATERIAL RECONCILIATION (/qs/material-recon)
----------------------

PURPOSE: Cross-references work done (from MB) against materials consumed
(from Store Issue Notes). This is the primary tool for detecting site
leakage, theft, or wastage.

WHAT YOU SEE:
  - Project selector
  - Reconciliation table with columns:
      * BOQ Item
      * Work Done Quantity (from MB)
      * Theoretical Consumption (calculated using standard coefficients)
      * Actual Issue (from MIN/Store records)
      * Variance (Actual - Theoretical)
      * Variance % (percentage deviation)
  - Items with high variance are highlighted in red

HOW TO READ:
  - If Variance is POSITIVE: More material was issued than theoretically
    needed. This indicates potential wastage or theft.
  - If Variance is NEGATIVE: Less material was issued. Either work was
    done without proper material issue notes, or efficiency is high.
  - Target: Keep variance within ±5% for most items.

WORKFLOW:
  1. Select a project
  2. System auto-calculates theoretical consumption based on MB progress
  3. System pulls actual consumption from Store Issue Notes (MIN)
  4. Review the variance column
  5. Investigate any items with >10% variance
  6. Take corrective action (store audits, wastage reports)

----------------------
5.2  TDS REGISTER (/finance/tds)
----------------------

PURPOSE: Track Tax Deducted at Source under Section 194C of the Income
Tax Act. Applicable on all contractor/sub-contractor payments.

TDS RATES:
  - Companies (Pvt Ltd / Ltd):  2% of taxable amount
  - Individuals / HUF / Firms:  1% of taxable amount
  - Threshold: TDS applicable if single payment > ₹30,000
    or aggregate payments > ₹1,00,000 in a financial year

WHAT YOU SEE:
  - Register table with all TDS entries
  - Monthly summary of deductions
  - Challan payment tracking

----------------------
5.3  VENDOR PAYABLES (/finance/invoices)
----------------------

PURPOSE: Bill booking and payment tracking for vendor invoices.
Implements the 3-Way Match (PO vs GRN vs Invoice).

WHAT YOU SEE:
  - List of vendor bills with status tracking
  - Verification workflow

BILL BOOKING PROCESS:
  1. Vendor submits a physical/digital invoice
  2. Accounts team enters the bill into the system
  3. System performs 3-WAY MATCH:
     - Invoice Price vs PO Price (must match)
     - Invoice Quantity vs GRN Quantity (must match)
     - If mismatch detected, system flags for review
  4. TDS is automatically calculated and deducted
  5. GST Input Credit is recorded
  6. Bill status: Pending -> Verified -> Paid

8.4  PAYROLL PROCESSING (/hr/payroll)
----------------------

PURPOSE: Automated salary calculation based on attendance data,
including statutory deductions (PF, ESI, Professional Tax).

PAYROLL CALCULATION LOGIC:
  - Basic Wage = Days Present × Daily Rate
  - Overtime = OT Hours × (Daily Rate / 8 × 1.5)
  - Gross Salary = Basic + Overtime + Allowances

  STATUTORY DEDUCTIONS:
  - PF (Provident Fund):
      * Employee contribution: 12% of Basic
      * Employer contribution: 12% of Basic (13% including admin)
      * Applicable if Basic ≤ ₹15,000/month
  - ESI (Employee State Insurance):
      * Employee contribution: 0.75% of Gross
      * Employer contribution: 3.25% of Gross
      * Applicable if Gross ≤ ₹21,000/month
  - Professional Tax: As per state slab (varies)

  Net Salary = Gross - PF - ESI - PT

WHAT YOU SEE:
  - Payroll register with all workers
  - Month/Year selector
  - Columns: Name, Days Worked, Basic, OT, Gross, PF, ESI, PT, Net
  - Total payroll summary
  - Export to CSV for bank upload

16.  SECURITY FEATURES
================================================================================

IMPLEMENTED SECURITY MEASURES:

  1. PASSWORD COMPLEXITY:
     Minimum 8 characters with uppercase, lowercase, number, and symbol.

  2. JWT TOKEN LIFECYCLE:
     Access tokens expire after 1 hour. Users must re-authenticate.

  3. GLOBAL IDLE TIMEOUT:
     60 minutes of inactivity (no mouse/keyboard) triggers automatic
     logout. Prevents unauthorized access on unattended workstations.

  4. INPUT VALIDATION:
     All API endpoints use express-validator to sanitize inputs and
     prevent SQL injection and XSS attacks.

  5. PASSWORD HASHING:
     bcrypt with 12 salt rounds. Passwords are never stored in plain text.

  6. DIGITAL SIGNATURES:
     Every approval action records the user's uploaded signature,
     timestamp, and IP address for forensic accountability.

  7. AUTOMATED BACKUPS:
     Database is backed up daily at 2:00 AM using pg_dump.
     Backups are retained for 7 days with automatic rotation.

17.  AUDIT TRAIL & LEAKAGE DETECTION GUIDE
================================================================================

HOW TO DETECT MATERIAL LEAKAGE:
  1. Go to QS > Material Reconciliation
  2. Select the project
  3. Review the Variance column:
     - Variance > +10%: Possible theft, wastage, or unauthorized use
     - Variance > +20%: Immediate investigation required
  4. Cross-reference with Store Ledger for physical stock count
  5. Check MIN records for unauthorized issues

HOW TO DETECT BILLING FRAUD:
  1. Go to Finance > Vendor Payables
  2. Run the 3-Way Match:
     - Compare Invoice Price vs PO Price
     - Compare Invoice Quantity vs GRN Quantity
  3. Any mismatch indicates potential fraud or error
  4. Check if the vendor GSTIN is valid
  5. Verify TDS has been correctly deducted

HOW TO DETECT MEASUREMENT INFLATION:
  1. Go to QS > Measurement Book
  2. Check photo attachments for geo-tagging
  3. Verify GPS coordinates match the project location
  4. Compare MB quantities against physical site progress
  5. Run RA Bill history to check for sudden quantity jumps

HOW TO DETECT PROCUREMENT ANOMALIES:
  1. Go to Procurement > Quotations & CS
  2. Verify that CS has at least 3 vendor quotes
  3. Check if the selected vendor is truly L1 (lowest)
  4. Review PO approval trail for any skipped stages
  5. Match PO quantities against MRS requirements
